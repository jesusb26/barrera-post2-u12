# ADR-001: Arquitectura Hexagonal para aislar el dominio

## Estado
Aceptado – 2026-05-27

## Contexto
El sistema de pedidos debe soportar múltiples tipos de procesamiento y canales de notificación. El acoplamiento directo del servicio a Spring Data JPA y a los mecanismos de notificación dificulta las pruebas unitarias y hace costoso cambiar la implementación de infraestructura.

## Decisión
Se adopta una arquitectura hexagonal (puertos y adaptadores). El dominio define puertos (interfaces como `ProcesadorPedido`). Los adaptadores (procesadores concretos, controladores, repositorios) implementan los puertos. El dominio no importa ninguna clase de Spring ni de JPA.

## Consecuencias
- **Positivas**: El dominio es testeable con Mockito sin levantar el contenedor Spring. Cambiar de JPA a MongoDB requiere solo un nuevo adaptador.
- **Negativas**: Aumenta el número de interfaces y clases de mapeo comparado con un MVC clásico. La curva de aprendizaje para el equipo es mayor.
