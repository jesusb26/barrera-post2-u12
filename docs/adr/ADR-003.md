# ADR-003: Spring Events (Observer) para notificaciones

## Estado
Aceptado – 2026-05-27

## Contexto
El sistema necesita notificar a múltiples receptores (email, log, futuro SMS) cuando un pedido es procesado. Acoplar directamente la lógica de cada canal dentro del servicio central lo hace frágil y difícil de extender.

## Decisión
Se utiliza `ApplicationEventPublisher` de Spring (implementación del patrón Observer) para publicar el evento `PedidoProcesadoEvent`. Cada canal de notificación se implementa como un `@EventListener` independiente, desacoplado del servicio principal.

## Consecuencias
- **Positivas**: Agregar un nuevo canal solo requiere crear un nuevo listener; no se modifica la fachada. Los listeners son opcionales: si no hay suscriptores, el evento se ignora sin errores.
- **Negativas**: El orden de ejecución de los listeners no está garantizado. El debugging es más complejo, pues hay que rastrear los listeners para entender qué ocurre tras publicar un evento.
